# Amazon S3

Amazon Simple Storage Service (S3) is AWS's object storage service, designed for high durability, scalability, and availability.

Unlike Amazon EBS, which provides block storage for EC2 instances, Amazon S3 stores data as objects and can be accessed independently by multiple AWS services.

This document introduces the core concepts of Amazon S3, including common use cases, storage classes, and lifecycle management.

---

## Common Use Cases

Amazon S3 is widely used for:

- Backups
- Static website hosting
- Media storage
- Application logs
- Data archiving
- Data lakes

---

## Storage Classes

Amazon S3 provides multiple storage classes to balance cost and access frequency.

### S3 Standard

Designed for frequently accessed data while providing high durability and availability.

### S3 Standard-Infrequent Access (Standard-IA)

Designed for infrequently accessed data that still requires low-latency retrieval.

### S3 One Zone-Infrequent Access (One Zone-IA)

Stores data in a single Availability Zone, offering lower cost in exchange for reduced resilience.

### S3 Glacier

Designed for long-term archival storage with very low storage costs and longer retrieval times.

---

## Lifecycle Management

S3 Lifecycle policies automatically transition objects between storage classes based on predefined rules.

A common lifecycle workflow is:

- Store new objects in S3 Standard.
- Move older objects to Standard-IA after a period of inactivity.
- Archive infrequently accessed objects in Glacier.

Lifecycle policies help optimize storage costs without requiring manual intervention.

---

## EBS vs. S3

Although both services store data, they serve different purposes.

| Amazon EBS | Amazon S3 |
|------------|-----------|
| Block storage | Object storage |
| Attached to EC2 instances | Independent AWS service |
| Low-latency disk access | Highly scalable object storage |
| Stores operating systems and application data | Stores files, backups, media, logs, and archives |
